# Roadmap Prodotto

Questa roadmap raccoglie le priorità attuali di `DocMolder`.

## Fase 1: Qualità e Affidabilità

- rendere la conversione in scala di grigi più coerente anche senza `Ghostscript`
- ampliare i test automatici su PDF reali, corrotti, pesanti o con struttura insolita
- migliorare messaggi e fallback quando una trasformazione non preserva bene il PDF nativo
- aggiungere monitoraggio più chiaro su errori e volumi d'uso

## Fase 2: Operatività e Controllo

- report admin periodici con riepilogo giornaliero o settimanale
- metriche più leggibili su utilizzo, errori e job falliti
- strumenti admin per capire rapidamente quali trasformazioni causano più problemi
- politiche di retention e manutenzione più visibili anche lato documentazione

## Fase 3: Espansione Funzionale

- raffinare ulteriormente compressione e scala di grigi sui PDF nativi
- valutare nuove trasformazioni PDF davvero utili e coerenti col prodotto
- migliorare la gestione dei casi documento più complessi senza peggiorare la semplicità d'uso

## Fase 4: Evoluzione Prodotto

- decidere se restare su un tool essenziale o aprire una nuova fase di crescita
- valutare se introdurre funzioni premium o solo miglioramenti di qualità
- definire se servono analytics più mature o automazioni admin più forti

## Fase 5: Funzioni Oggi Fuori Scope

Questa sezione raccoglie idee oggi escluse dal perimetro del progetto, ma che possono tornare utili se in futuro cambiano priorità o direzione del prodotto.

- supporto gruppi Telegram
- dashboard web amministrativa
- storage permanente dei file utente
- webhook e infrastruttura HTTP pubblica
- storico lavori per l'utente
- funzioni PDF avanzate come watermark, firma, estrazione pagine o editing avanzato
- OCR
- UI web
